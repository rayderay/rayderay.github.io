JavaScript执行顺序

JavaScript运行机制

JavaScript是一种描述型脚本语言，它不需要进行编译成中间语言，而是由浏览器进行动态的解析与执行。

代码块

  1. JavaScript中的代码块是指由script标签分割的代码段
  2. JavaScript是按照代码块来进行编辑和执行的，代码块间相互独立，但变量和方法共享。

声明式函数与赋值式函数

声明式函数与赋值式函数的区别在于：在JS的预编译期，声明式函数将会被先提取出来，然后才按照顺序执行JS代码

预编译期与执行期

  1. 预编译期：JS会对本代码块中的所有声明的变量和函数进行处理。但需要注意的是此时处理函数的只是声明式函数，而且变量也只是进行了声明但是未进行初始化以及赋值。

需要注意的点是，JS引擎是按照代码块来顺序执行的，其实完整的说法应该是按照代码块来预执行和处理，也就是说预处理的只是执行到的代码块的声明函数和变量，对于未加载到的代码块，是没有办法进行预处理的。

执行顺序

  1. 读入第一个代码块。
  2. 做语法分析，有错则报语法错误，同时并跳转到step5
    3.对var变量和function定义做“预编译处理，这项操作永远不会报错，因为只解析正确的声明。
  3. 执行代码段，有错则报错（比如变量未定义）
  4. 如果还有下一个代码段，则读入下一个代码段，重复step2
  5. 结束
