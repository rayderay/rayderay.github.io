webpack基本配置

什么是webpack

webpack是前端模块化工具。可以找到JavaScript模块以及其他一些浏览器不能直接运行的扩展语言（sass,Typescript等），并将其转换和打包为合适的格式供浏览器使用

webpack工作方式

把项目当做整体，通过一个给定的入口主文件，将从这个文件开始找到项目的所有的依赖文件，处理之后打包为一个（或者多个）浏览器可识别的JavaScript文件

webpack主要配置文件

在项目目录底下创建一个命名为webpack.config.js的文件。这个配置文件也是一个简单的JavaScript模块。

entry

入口文件，可以是一个入口文件，也可以使用对象的形式设置多个入口文件

output

出口文件。将所有文件打包成输出的这样的文件。如果是多个文件，即生成多个文件。

module

通过使用loader，webpack有能力调用外部的工具，实现对不同格式文件的处理。

test:
一个用以匹配loaders所处理文件的扩展名的正则表达式（必须）

include/exclude:手动添加必须处理的文件（文件夹）或屏蔽不需要处理的文件（可选）

query: 为loaders提供额外的设置选项

Plugins

常用的插件有

HtmlWebpackPlugin

该插件的基本作用就是生产HTML

将webpack配置项中entry的相关入口thunk和利用‘extract-text-webpack-plugin“抽取的CSS样式，插入到该插件提供的template或者templateContent配置项指定的内容基础上生成一个HTML文件。

查看该插件提供的配置项如下：

  • filename：输出文件的文件名称，默认为index.html,不配置就是该文件名;此外，还可以为输出文件指定目录位置。

    1. filename配置的HTML文件目录是相对于webpackConfig.output.path路径而言的，不是相对于当前项目目录结构的。
    2. 制定生成的HTML文件内容中的link和script路径是相对于生成目录下的，写路径的时候需要写成生成目录下的相对路径。

  • template : 本地模板文件的位置，支持加载器（如handlebars,ejs,underscore,html),如handlebars！src/index.hbs

  • inject :向template或者templateContent中注入所有静态资源，不同的配置值注入的位置不相同

  1. true/body:所有JavaScript资源插入到body元素的底部
  2. head:所有的JavaScript资源插入到head元素中
  3. false:所有的静态资源css和JavaScript都不会注入到模板文件中
